# 開發紀錄 - 2026-04-17

## 本次重點：`data_access`（DAC）與後台入口邏輯、移除敏感資料「解鎖／遮罩」管線

本次變更將「資料範圍（Z 軸）」從 **Role 文件上的 `scope`** 抽離，改為獨立 Firestore 集合 **`data_access/{userId}`**；功能權限（X 軸）仍由 **Role 權限聯集** 決定。並同步 **下架會友敏感欄位的遮罩／reveal API／稽核 stub** 等舊機制，會友 API 改為在通過 scope 檢查後直接回傳欄位內容。

> 統計（`git diff HEAD --stat`）：約 52 檔、`+1537 / -1403` 行（含新增 Access Control UI）。

---

## 1. 資料模型：`data_access` 與型別

- **新路徑**：`Firestore` `data_access/{userId}` 單一文件。
- **結構**（見 `app/types/data-access.ts`）：
  - **`admin`**：`isGlobal`、`zone[]`、`group[]` — 行政／組織資料範圍。
  - **`functions`**：`isGlobal`、`targets: Record<string, string[]>` — 功能面向的目標 ID（例如小組等功能群組）。
  - 附帶 `updatedAt`、`updatedBy`；PATCH 支援 `grant` / `revoke` / `setGlobal`。
- **`DataAccessRepository`**（`server/repositories/data_access.repository.ts`）：
  - 讀取時 **正規化** 舊版曾用 `set(merge)` 寫入的 **字面欄位**（如 `admin.isGlobal`）與 **nested** 並存問題；**nested 有 key 時優先**，避免關掉 global 後讀取仍為 `true` 的 bug。
- **`dacEngine`**（`server/utils/dacEngine.ts`）：
  - 對 `getDataAccess(userId)` 做 **5 分鐘記憶體快取**；`clearDacCache` 供寫入後失效。
  - 提供 `getAdminScope` / `getFunctionsTargetIds` 給後續 scope 解析使用。

## 2. API：`/api/data-access/:userId`

| 方法 | 檔案 | 行為概要 |
|------|------|-----------|
| `GET` | `index.get.ts` | 讀取該使用者的 `data_access`（無則視為 null／空） |
| `PUT` | `index.put.ts` | 整份覆寫（`SetDataAccessPayload`） |
| `PATCH` | `index.patch.ts` | 細粒度 grant/revoke/setGlobal；成功後 **`AuthService.clearCache(userId)`** |
| `DELETE` | `index.delete.ts` | 刪除文件 |

- **授權**：PATCH/PUT/DELETE 等變更需 **`requireAbility(event, "manage", "System")`**（與後台「系統／角色」同一等級的管理員能力）。

## 3. 認證上下文：`UserContext` 與 `resolveUserContext`

- **`app/types/auth.ts`**：`UserContext` 新增 **`accessScope: { admin, functions }`**，由 `data_access` 填入；缺文件時為空 admin／空 functions（等同僅 Self 範圍，依各 Service 的 scope 工具行為而定）。
- **`server/utils/rbac/resolver.ts`**：
  - `resolveUserContext(member, roles, dataAccess)` — **權限仍為多角色聯集**；**資料範圍只信 `data_access`**，不再從 role 合併 scope。
- **`server/services/auth.service.ts`**：
  - `resolveContext` 內 **`getDataAccess(userId)`** 後組裝 context；`clearCache` 時同步清 **context cache + DAC cache**。

- **`applyScopeConstraints` / `assertScopeAccess` / `filterByScope`** 支援第二參數 **`scopeType: "admin" | "functions"`**（預設 `admin`）。
